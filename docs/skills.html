Game Project: Maze Runner

Problem Solving in Game Development

During my programming module, I created a maze game using C++ and SFML — a library I had no prior experience with. Challenges included developing a working GUI system and implementing a random maze generator. I also added a login system, scoring, a game-over state, and a timer. These demonstrate my ability to quickly learn new tools and combine systems effectively — an essential skill in game and software development.

Self-Teaching SFML

SFML wasn't taught during lectures, so I independently followed tutorials and forums to learn how to use it. Understanding the event loop structure was tricky at first, but I gradually adapted to it and overcame rendering issues through further research.

Debugging and Testing

I regularly tested my game to make sure all features worked. This included verifying the login system, making sure timers and scores updated correctly, and testing edge cases like timeouts or rapid menu clicks. Through this, I developed a better understanding of how my code interacted and became quicker at spotting bugs.
